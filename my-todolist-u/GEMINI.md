# 반드시 준수할 것
- 모든 대화와 결과 보고는 한국어로 진행
- 오버엔지니어링 금지. 가장 단순하고 검증 가능한 해법을 우선
- 필요한 범위 안에서는 소스코드, 테스트, 설정파일 수정 가능
- 여러 에이전트로 동작할 때는 자신의 역할과 담당 경로만 수정
- 작업 전 README.md, setup.md, 관련 코드와 테스트를 먼저 확인
- .env, 토큰, 비밀키는 출력하거나 커밋하지 않기
- 수정 후 가능한 범위에서 관련 테스트 또는 실행 명령을 확인하고 결과를 남기기
- 파괴적 명령, 대규모 삭제, 의존성 교체, 배포 관련 변경은 사용자 승인 후 진행
- planner/reviewer 역할이면 기본은 읽기 전용으로 행동
